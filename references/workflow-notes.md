# Workflow Notes

## Use Cases

- Journal asks for a revision manuscript where changed text is highlighted.
- The manuscript has Track Changes and the user wants inserted text highlighted in a clean copy.
- The user already added manual highlights and wants to add revision highlights without removing them.
- The user wants deleted passages represented visibly as red strikethrough text.
- The user forgot to enable Track Changes but has an older file and the latest file.

## Reconstructing Track Changes

When no Track Changes file exists, use Word's compare feature:

1. Open Word.
2. Choose Review > Compare > Compare Documents.
3. Set the original document to the old version.
4. Set the revised document to the latest version.
5. Leave the reviewer name blank if it does not matter.
6. Save the compared document, then run the macro on that compared document.

## Macro Behavior

The default macro generated by `scripts/generate_macro.py`:

- Turns off recording of new revisions before formatting.
- Requires the active document to be saved.
- Creates a new document from the active file so the source file is not modified directly.
- Clears existing highlights in the new document unless `--preserve-existing-highlights` is used.
- Highlights inserted revisions with the chosen Word highlight color.
- Accepts revisions after marking unless `--keep-revisions` is used.

## Deletions

If the user wants deleted text to remain visible, use `--mark-deletions`. This rejects deletion revisions, bringing deleted text back into the output, then formats it with red strikethrough.

Do not combine `--mark-deletions` with `--keep-revisions`: restoring deleted text necessarily resolves those deletion revisions.

## Color Constants

Use Microsoft Word `WdColorIndex` values with `HighlightColorIndex`.

- `wdYellow`: yellow
- `wdBrightGreen`: bright green
- `wdTurquoise`: turquoise
- `wdPink`: pink
- `wdGray25`: 25 percent gray
- `wdGray50`: 50 percent gray
- `wdNoHighlight`: no highlight

## Cautions

- Ask the user to keep a backup or work on a copy before running VBA.
- Word macro security settings may block execution until macros are enabled for the document.
- If an editing company accepted old revisions and left only new proofing revisions, ask them not to remove revision history next time, or use a pre-highlighted copy before sending it out.
- Field codes, comments, tables, and reference managers can behave differently across Word versions. Recommend spot-checking the output before submission.
